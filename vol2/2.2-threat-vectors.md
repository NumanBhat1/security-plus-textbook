## 3. Learning Outcomes

After this objective you will be able to:

1. Distinguish a **threat vector** (the path in) from an **attack surface** (the sum of exposed points).
2. Identify the technical vectors: message/image/file-based, voice, removable devices, vulnerable/unsupported software, unsecure networks, open ports, default credentials, supply chain.
3. Recognize all the **human/social-engineering** vectors (phishing, vishing, smishing, pretexting, watering hole, typosquatting, BEC, etc.).
4. Match a scenario to the correct vector — CompTIA's main format here.

---

## 4. Concept Overview

A **threat vector** (or *attack vector*) is the **method or pathway** an attacker uses to gain access — the *how they get in*. The **attack surface** is the **total set of points** where an attacker could try to enter (every exposed service, device, user, and integration). Reducing the attack surface and closing vectors is the core of prevention.

This objective is a big list of *ways in*, split into two halves:

1. **Technical vectors** — messages, files, images, voice, removable media, vulnerable/unsupported software, unsecure networks, open ports, default creds, and the supply chain.
2. **Human vectors (social engineering)** — the many flavors of tricking people (phishing family, pretexting, watering hole, typosquatting, BEC, impersonation, mis/disinformation).

The social-engineering terms are *heavily* tested and easy to confuse (vishing vs. smishing vs. phishing), so they get extra attention here.

## 6. Deep Technical Explanation

### Threat vector vs. attack surface

- **Attack surface** = *all* the doors and windows of the house (every exposed service, port, device, user, integration).
- **Threat vector** = the *specific door* an attacker walks through (e.g., a phishing email).

Shrinking the attack surface (closing ports, removing software, fewer entry points) reduces the number of vectors available.

### Technical vectors

| Vector | How it's abused |
|--------|-----------------|
| **Email** | Phishing, malicious links/attachments — the #1 vector. |
| **SMS** | Smishing (texts with malicious links). |
| **IM** | Malicious links/files via chat apps. |
| **Image-based** | Malicious payloads embedded in images (steganography, malformed images exploiting parsers). |
| **File-based** | Macro-laden documents, weaponized PDFs, trojanized executables. |
| **Voice call** | Vishing — phone calls to trick victims (fake support, fake bank). |
| **Removable device** | Malicious USBs (USB drop attacks), data exfiltration. |
| **Vulnerable software** | Exploiting flaws — **client-based** (installed app/agent) vs. **agentless** (browser/web service with no installed client). |
| **Unsupported systems** | End-of-life OS/apps that no longer get patches = permanent holes. |
| **Unsecure networks** | **Wireless** (open/weak Wi-Fi), **wired** (unsecured ports), **Bluetooth** (bluejacking/bluesnarfing). |
| **Open service ports** | Listening ports expose services; each open port is a potential way in. |
| **Default credentials** | admin/admin left unchanged — trivial access. |
| **Supply chain** | Compromise via **MSPs, vendors, suppliers** (a trusted third party becomes the path in). |

> **Client-based vs. agentless** is a frequent exam point: client-based needs installed software (an agent) that can itself be vulnerable; agentless has no installed client (e.g., accessed through a browser), shifting the risk surface.

### Human vectors / social engineering (heavily tested)

The **phishing family** — distinguished by *delivery channel*:

| Term | Channel | Memory anchor |
|------|---------|---------------|
| **Phishing** | Email | the general term (email) |
| **Vishing** | **V**oice / phone | "**V** = Voice" |
| **Smishing** | **SMS** / text | "**SM** = SMS" |

Other social-engineering vectors:

| Vector | What it is |
|--------|-----------|
| **Pretexting** | Inventing a believable scenario ("I'm from IT, I need your password to fix your account"). |
| **Impersonation** | Posing as a trusted person (exec, help desk, vendor). |
| **Business email compromise (BEC)** | Spoofing/hijacking a real business account to request fraudulent wire transfers or data — often impersonating an executive ("CEO fraud"). |
| **Watering hole** | Infecting a website the target *group* is known to visit, so victims compromise themselves. |
| **Brand impersonation** | Faking a trusted brand (logos, lookalike emails/sites) to deceive. |
| **Typosquatting** | Registering lookalike/misspelled domains (e.g., `gooogle.com`) to catch typos. |
| **Misinformation / disinformation** | Spreading false info — *mis* = unknowingly false; *dis* = deliberately false (influence operations). |

> **Social engineering exploits human psychology** — authority, urgency, trust, fear, scarcity (more in 2.4 / awareness 5.6). The defense is awareness training, reporting, and technical filters.

### Why these are grouped

All of these are **ways to reach a victim or system**. CompTIA wants you to read a scenario and name the vector: a text with a malicious link = smishing; a phone call pretending to be the bank = vishing; a misspelled domain = typosquatting; a fake wire-transfer email from "the CEO" = BEC.

---

## 7. Why It Exists

Every attack needs a way in. This objective exists so defenders can **inventory and reduce their attack surface** and **harden the most-used vectors** (email above all). You can't defend what you haven't enumerated — knowing the vectors lets you map controls (email filtering, port closure, patching, awareness training) to each path.

---

## 8. How It Works (enterprise)

Security teams continuously **map the attack surface** (asset inventory, external scans, exposed services) and **close vectors**: email security gateways and phishing training for message-based; USB/device control for removable media; patch/EOL management for vulnerable/unsupported software; network segmentation and Wi-Fi hardening for unsecure networks; port scanning and firewalls for open ports; credential policies for default creds; and vendor risk management (5.3) for the supply chain. Human vectors are addressed by awareness programs (5.6) plus technical filters.

---

## 9. Advantages (of understanding vectors/surface)

- **Targeted hardening** — fix the paths attackers actually use.
- **Attack-surface reduction** — fewer exposed points = fewer vectors.
- **Better detection** — knowing the vector sharpens monitoring.
- **Prioritization** — email and credentials get the most attention because they're the most-abused.

---

## 10. Disadvantages / Challenges

- **Surface grows constantly** — new apps, cloud, IoT, remote work expand it.
- **Human vectors are hard to fully close** — people remain trickable.
- **Supply chain is largely outside your control** — you depend on others' security.
- **Shadow IT/unknown assets** create vectors you can't see.

---

## 11. Real-World Examples

- **Email phishing** delivering ransomware via a malicious attachment.
- **Smishing:** "Your package is held — click to pay a fee" text.
- **Vishing:** a caller impersonating IT support asking for an MFA code.
- **BEC:** a spoofed CEO email instructing finance to wire funds to a new account.
- **Watering hole:** a compromised industry forum infecting all visitors.
- **Typosquatting:** `paypa1.com` harvesting credentials.
- **Removable device:** USB drives dropped in a parking lot.
- **Supply chain:** a compromised software update from a trusted vendor.

---

## 12. Enterprise Usage (SOC relevance)

- **Email is your #1 alert source** — phishing triage (your phishing-header analyzer) is daily SOC work.
- **Open-port / external-surface findings** feed vulnerability management.
- **Smishing/vishing** reports come in via users — you investigate and advise.
- **Supply-chain alerts** (vendor breach notifications) become incidents.
- Recognizing the **vector** in an alert shapes containment (block sender, isolate USB host, reset creds).

---

## 13. Security Implications

- **Default credentials and open ports** are trivially exploited — among the most common root causes.
- **Unsupported/EOL systems** are permanent, unpatchable holes.
- **Human vectors** bypass technical controls entirely (a user hands over the keys).
- **Supply-chain compromise** can affect many victims at once and is hard to detect.
- **Unsecure Bluetooth/Wi-Fi** exposes nearby attack opportunities.

---

## 14. Attacker / Adversary Perspective

- Attackers pick the **cheapest reliable vector** — usually email/social engineering.
- They **scan for open ports and default creds** as low-hanging fruit.
- They target **EOL systems** knowing patches will never come.
- They use **typosquatting and brand impersonation** to ride trust.
- **Supply chain** is the force-multiplier vector — one compromise, many victims.

---

## 15. Defender Perspective

- **Reduce the attack surface:** close ports, remove unused software, retire EOL systems.
- **Harden the top vectors:** email security, MFA (kills many credential attacks), device control.
- **Change default credentials everywhere** (a hardening must — 2.5).
- **Segment networks** and secure wireless/Bluetooth.
- **Manage supply-chain risk** (5.3) and **train users** (5.6).

---

## 16. Detection Techniques

- **Email security gateways / anti-phishing** + user reporting.
- **External attack-surface scanning** (open ports, exposed services).
- **EDR** for file/image/removable-media payloads.
- **Network monitoring** for anomalous wireless/Bluetooth/connections.
- **DNS monitoring** for typosquatting/known-bad domains.

---

## 17. Mitigation Techniques

- **Email filtering, link rewriting, attachment sandboxing.**
- **Patch + EOL management** (kill unsupported-software vectors).
- **Close unused ports; firewall; segment.**
- **Change default credentials; enforce strong creds + MFA.**
- **USB/device control; disable unneeded Bluetooth.**
- **Vendor risk management; awareness training.**

---

## 18. Best Practices

1. **Inventory and continuously reduce your attack surface.**
2. **Prioritize email and credential defenses** — the most-abused vectors.
3. **Eliminate default credentials and close unused ports.**
4. **Retire/segment unsupported (EOL) systems.**
5. **Control removable media** and disable unneeded wireless/Bluetooth.
6. **Manage supply-chain/vendor risk** (5.3).
7. **Train users** on the social-engineering vectors (5.6).

## 23. Interview Notes

- *"Vector vs. attack surface?"* — Vector is the specific path used; attack surface is the sum of all exposed entry points.
- *"What's BEC?"* — Business email compromise: fraud via spoofed/hijacked business email, often impersonating an executive to request wire transfers.
- *"Phishing vs. vishing vs. smishing?"* — Email vs. voice vs. SMS delivery.
- *"What's a watering hole attack?"* — Compromising a site the target group frequents so victims infect themselves.
- *"Why are default credentials dangerous?"* — They're publicly known and trivially exploited if unchanged.

## 24. Common Beginner Mistakes

1. **Swapping vishing and smishing** (voice vs. SMS).
2. **Confusing pretexting (a story) with impersonation (a person).**
3. **Treating attack surface and vector as the same thing.**
4. **Forgetting BEC is email-based fraud**, not generic phishing.
5. **Overlooking default credentials/open ports** as major vectors.
6. **Mixing mis- and disinformation** (unknowing vs. deliberate).
7. **Forgetting EOL/unsupported software is a vector** (no patches ever).

## 29. Practice Questions

**Q1.** An employee receives a text message claiming a package is held and to "click here to pay." This vector is:
A. Phishing  B. Vishing  C. Smishing  D. Pretexting

**Q2.** A finance employee receives an urgent email appearing to come from the CEO requesting a wire transfer to a new account. This is:
A. Watering hole  B. Business email compromise (BEC)  C. Typosquatting  D. Smishing

**Q3.** Which term describes the TOTAL set of points where an attacker could attempt to gain access?
A. Threat vector  B. Attack surface  C. Payload  D. Exploit

**Q4.** Attackers compromise a website frequently visited by employees of a target company, infecting those visitors. This is a:
A. Watering hole attack  B. Vishing attack  C. Supply chain attack  D. Pretexting attack

**Q5.** A user mistypes a popular website's address and lands on a malicious lookalike site at a misspelled domain. The attacker used:
A. Brand impersonation  B. Typosquatting  C. Pretexting  D. BEC

**Q6.** A caller pretends to be from the IT help desk and invents a story to convince a user to reveal their password. This combines impersonation with:
A. Smishing  B. Pretexting  C. Watering hole  D. Typosquatting

**Q7.** Which vector involves end-of-life software that no longer receives security patches?
A. Open service ports  B. Default credentials  C. Unsupported systems and applications  D. Removable device

**Q8.** A network device still uses the manufacturer's admin/admin login. This is a vector known as:
A. Default credentials  B. Agentless software  C. Watering hole  D. Disinformation

**Q9.** Which best distinguishes client-based from agentless software?
A. Client-based runs in the cloud; agentless runs locally
B. Client-based requires installed software (an agent); agentless does not
C. They are identical
D. Agentless is always more secure

**Q10.** Deliberately spreading false information to influence or deceive is:
A. Misinformation  B. Disinformation  C. Pretexting  D. Impersonation

### Answer Key & Explanations

- **Q1 — C (Smishing).** Malicious SMS.
- **Q2 — B (BEC).** Spoofed exec email requesting a wire = business email compromise.
- **Q3 — B (Attack surface).** All exposed entry points.
- **Q4 — A (Watering hole).** Compromise a site the target visits.
- **Q5 — B (Typosquatting).** Misspelled lookalike domain.
- **Q6 — B (Pretexting).** An invented scenario (plus impersonation).
- **Q7 — C (Unsupported systems).** EOL = no more patches.
- **Q8 — A (Default credentials).** Unchanged factory login.
- **Q9 — B.** Client-based needs an installed agent; agentless doesn't.
- **Q10 — B (Disinformation).** Deliberately false (mis- is unknowing).
