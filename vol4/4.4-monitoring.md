---

## 4. Concept Overview

Monitoring is the **eyes of security operations** — continuously watching systems, applications, and infrastructure, collecting their data, and **alerting** on anything suspicious. This is the heart of the SOC analyst job and overlaps your Priority 7.

The spine: **collect data → aggregate → alert → respond → tune.** Then know the **tools** that do it, especially the **SIEM** (the central correlation engine) and the data sources that feed it (logs, NetFlow, SNMP).

---

## 5. Definitions

| Term | Definition |
|------|------------|
| **Log aggregation** | Centralizing logs from many sources. |
| **Alerting** | Notifying when something suspicious is detected. |
| **Archiving** | Storing logs long-term (compliance/investigation). |
| **Alert tuning** | Adjusting rules to reduce false positives/noise. |
| **Quarantine** | Isolating a suspicious file/host. |
| **SIEM** | Security Information and Event Management — central log **collection + correlation + alerting**. |
| **SCAP** | Security Content Automation Protocol — standardizes security/config checking. |
| **Benchmark** | A secure-config standard (e.g., CIS Benchmarks). |
| **Agent-based** | Monitoring via installed software on the host. |
| **Agentless** | Monitoring without installed software (remote/protocol). |
| **Antivirus** | Detects/removes malware (signature + heuristic). |
| **DLP** | Data Loss Prevention — stops sensitive data from leaving. |
| **SNMP trap** | An alert pushed by a device to a manager when an event occurs. |
| **NetFlow** | Traffic **metadata** (who talked to whom, ports, volume). |
| **Vulnerability scanner** | Finds known weaknesses. |

## 6. Deep Technical Explanation

### What to monitor

- **Systems** — OS/host health, security logs, processes.
- **Applications** — app logs, errors, anomalies.
- **Infrastructure** — network devices, traffic, cloud.

### Activities

| Activity | What it is |
|----------|-----------|
| **Log aggregation** | Centralize logs (→ SIEM) so they can be correlated |
| **Alerting** | Fire notifications on suspicious events |
| **Scanning** | Vulnerability/config scans |
| **Reporting** | Summaries/metrics for stakeholders |
| **Archiving** | Long-term log storage (compliance/forensics) |
| **Alert response/validation** | **Quarantine** the threat; **tune** alerts to cut noise |

> **Alert tuning** is critical: too many false positives → alert fatigue → real alerts missed. Tuning adjusts thresholds/rules. **Quarantine** isolates the suspicious file/host while you investigate.

### Tools (know each one's job)

| Tool | Job | Tell |
|------|-----|------|
| **SIEM** | **Aggregates + correlates logs**, alerts | "centralized log correlation" |
| **SCAP** | Standard protocol for **automated config/vuln checking** | "standardized security automation" |
| **Benchmarks** | Secure-config **standards** (CIS) | "config baseline standard" |
| **Agent-based** | Monitor via **installed software** | "agent installed on host" |
| **Agentless** | Monitor **without installed software** | "no agent, remote/protocol" |
| **Antivirus** | Detect/remove **malware** | "signature/heuristic malware detection" |
| **DLP** | Stop **sensitive data leaving** | "blocks data exfiltration/leakage" |
| **SNMP trap** | Device **pushes an alert** on an event | "device sends an alert/notification" |
| **NetFlow** | **Traffic metadata** (flows) | "who talked to whom, bandwidth" |
| **Vulnerability scanner** | **Find weaknesses** | "identifies vulnerabilities" |

### The key distinctions (your Priority 7)

- **SIEM vs NetFlow:** SIEM **correlates logs** from everywhere; NetFlow is **traffic metadata** (one input — flow records, not packet contents). "Centralized log correlation" = SIEM; "traffic flow data, who-talked-to-whom" = NetFlow.
- **SNMP vs SNMP trap:** SNMP **polls** devices for status (manager asks); an **SNMP trap** is **pushed by the device** when an event occurs (device tells you, unsolicited).
- **Agent vs agentless:** agent = **installed software** (deeper visibility, more overhead); agentless = **no install** (lighter, less depth).
- **DLP vs antivirus:** DLP watches **data leaving**; antivirus watches for **malware**.
- **SIEM vs SOAR (from 4.7):** SIEM **detects/alerts**; SOAR **automates the response**.

> **SNMP poll vs trap:** poll = manager **asks** ("are you OK?"); trap = device **tells** ("something happened!"). Trap = pushed/unsolicited.

## 7. Why It Exists

Prevention isn't perfect — you must **detect** what gets through. Monitoring exists to give the SOC continuous visibility, turning the flood of system/app/network data into actionable alerts. The SIEM exists because correlating events across many sources reveals attacks that any single log would miss.

---

## 8. How It Works (enterprise)

Hosts, apps, and infrastructure send logs (and NetFlow, SNMP traps) to a **SIEM**, which **aggregates and correlates** them and **alerts** on rules/anomalies. Analysts triage alerts, **quarantine** threats, and **tune** rules to cut noise. **Vulnerability scanners** and **SCAP/benchmarks** check configs; **DLP** watches data egress; **antivirus/EDR** covers endpoints. Logs are **archived** for compliance/forensics.

---

## 9. Advantages

- **Visibility + early detection** (SIEM correlation).
- **Compliance** (log archiving/retention).
- **Faster response** (alerting + quarantine).
- **Less noise** (alert tuning).
- **Broad coverage** (systems + apps + infrastructure).

---

## 10. Disadvantages / Challenges

- **Alert fatigue** (too many false positives → tuning needed).
- **Cost/complexity** (SIEM tuning and storage).
- **Data volume** (logs are huge → archiving cost).
- **Coverage gaps** (un-onboarded sources are blind spots).

---

## 11. Real-World Examples

- **SIEM correlation:** failed logins + new admin account + data transfer → correlated into one incident.
- **SNMP trap:** a switch sends a trap when an interface goes down.
- **NetFlow:** a host suddenly sends gigabytes to an unknown external IP (possible exfil).
- **DLP:** blocks an email with a spreadsheet of SSNs leaving the org.
- **Alert tuning:** a noisy rule generating 500 daily false positives is refined.

---

## 12. Enterprise Usage (SOC relevance)

This **is** the SOC analyst's environment:
- You live in the **SIEM**, triaging correlated alerts.
- You read **logs, NetFlow, and SNMP traps** as data sources.
- You **quarantine** threats and **tune** noisy alerts.
- **DLP/antivirus/scanner** outputs are events you investigate.
- **Archiving** supports your investigations (4.9).

---

## 13. Security Implications

- **No monitoring** = breaches undetected for months.
- **Poor tuning** = alert fatigue → missed real attacks.
- **Missing log sources** = blind spots.
- **Short archiving** = no data for investigation/compliance.
- **NetFlow spikes** are early exfil/DDoS signals.

---

## 14. Attacker / Adversary Perspective

- Attackers try to **avoid generating alerts** (low-and-slow, living off the land).
- They **clear or disable logs** (missing logs = anti-forensics — 2.4).
- They blend into **normal traffic** to evade NetFlow/SIEM detection.
- They exploit **un-monitored** systems (blind spots).

---

## 15. Defender Perspective

- **Onboard all log sources** to the SIEM; correlate.
- **Tune alerts** to balance noise vs coverage.
- **Monitor NetFlow** for anomalies (exfil/DDoS).
- **Use SNMP traps** for device events.
- **Archive logs** per retention; **protect log integrity**.
- **Layer DLP + antivirus/EDR + scanners.**

---

## 16. Detection / Tooling

- **SIEM** (correlation/alerting) — the hub.
- **NetFlow/sFlow/IPFIX** for traffic patterns.
- **SNMP traps** for device events.
- **DLP** for data egress; **antivirus/EDR** for malware.
- **SCAP/benchmarks** for config compliance.
- **Vulnerability scanners** for weaknesses.

---

## 17. Mitigation / Response Techniques

- **Quarantine** suspicious hosts/files.
- **Alert tuning** to reduce false positives.
- **Block/remediate** via integrated tools.
- **Archive** for forensics/compliance.

---

## 18. Best Practices

1. **Centralize logs in a SIEM** and correlate.
2. **Tune alerts** continuously (fight alert fatigue).
3. **Monitor systems + apps + infrastructure** (full coverage).
4. **Use NetFlow** for traffic anomalies; **SNMP traps** for device events.
5. **Deploy DLP + antivirus/EDR + scanners.**
6. **Archive logs** per retention; protect their integrity.

## 21. Exam Tips

- **SIEM = centralized log aggregation + correlation + alerting.**
- **NetFlow = traffic metadata** (flows: src/dst/port/volume) — NOT packet contents, NOT log correlation.
- **SNMP trap = a device pushes an alert** (poll = manager asks).
- **Agent-based = installed software; agentless = no install.**
- **DLP = stops sensitive data leaving; antivirus = malware.**
- **SCAP = standardized automated security/config checking; benchmarks (CIS) = the config standards.**
- **Alert tuning = reduce false positives** (fights alert fatigue).
- **Quarantine = isolate a suspicious file/host.**
- **SIEM detects/alerts; SOAR automates the response (4.7).**
